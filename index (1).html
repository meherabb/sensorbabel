Abstract

Multimodal large language models (MLLMs) routinely hallucinate physical-world claims — inventing objects, misstating distances, and confabulating velocities — at rates between 20% and 35% on standard benchmarks. Existing fixes either patch the decoding process or require expensive retraining, and both approaches stay entirely inside the model, fighting its biases with more of its own statistics.

We take a different route. Sensor-rich platforms such as autonomous vehicles already carry the instruments that can directly adjudicate the claims MLLMs are most often wrong about. We formalize this idea as sensor authority: the mutual information between a claim's semantic type and a sensor modality's measurement, and prove that physically orthogonal sensors provide a hallucination firewall independent of the model's generation process.

Building on this, we introduce SensorBabel, a lightweight four-stage inference-time module — claim extraction, sensor routing, verdict scoring, and contrastive re-weighting — that wraps any frozen MLLM without touching its weights. Evaluated on SensorHalluc, a new benchmark of 4,200 synchronized multi-sensor driving scenes, SensorBabel reduces CHAIR S from 72.7 to 27.1 and improves SH-F1 by 34.9 points over an RLHF-finetuned baseline using only 30,218 trainable parameters.

Key Theoretical Guarantees

Theorem 3.3 — Sensor Independence Firewall

If sensor si is physically orthogonal to the MLLM's generation channel, then as sensor noise ‖εi‖ → 0, the probability of falsely suppressing a true claim → 0 independently of pM. LiDAR and a language model's token distribution share no statistical dependencies by construction.

Theorem 3.4 — Near-Bayes-Optimal Routing Regret

The empirical routing policy π̂ achieves regret O(√(k log k / N)). With k = 5 sensors the k log k term is small — explaining why 168 labeled scenes (5% of data) already reach within 1.1 CHAIR S points of full-data performance.

Theorem 3.5 — Corrected Entropy Bound

Under the SensorBabel corrected distribution (Eq. 9), output entropy satisfies H(pSB) ≤ H(pM) · exp(−λ·Δmin), concentrating the decoding distribution around sensor-consistent outputs.

sensor_routing_policy.py
12,933 params · Paper §4.3
# ── Stage 2: Sensor Routing Policy (SRP) ─────────────────────────────────
# π̂(cⱼ) = argmax_i SRP_ψ(eⱼ, a)_i
# Input: claim embedding + sensor availability vector
# Output: routing probability distribution over sensors

import torch
import torch.nn as nn
import torch.nn.functional as F

class SensorRoutingPolicy(nn.Module):
    """
    3-layer MLP routing policy trained with cross-entropy
    plus entropy regularisation (β=0.05) to prevent
    degenerate routing under partial sensor availability.

    Loss: L_SRP = L_CE - β · H(SRP_ψ(eⱼ, a))
    """

    def __init__(self,
                 input_dim: int = 69,   # CLAIM_DIM + N_SENSORS
                 hidden1:   int = 128,
                 hidden2:   int = 64,
                 n_sensors: int = 5):
        super().__init__()
        self.net = nn.Sequential(
            nn.Linear(input_dim, hidden1),
            nn.GELU(),
            nn.Linear(hidden1, hidden2),
            nn.GELU(),
            nn.Linear(hidden2, n_sensors),
        )

    def forward(self, claim_emb, avail):
        # claim_emb : (B, CLAIM_DIM)
        # avail     : (B, N_SENSORS) binary mask
        # returns   : (B, N_SENSORS) routing logits
        x      = torch.cat([claim_emb, avail], dim=-1)
        logits = self.net(x)
        # Mask unavailable sensors before softmax
        logits = logits.masked_fill(avail == 0, -1e9)
        return logits

    def route(self, claim_emb, avail):
        """Returns hard argmax routing decision."""
        with torch.no_grad():
            logits = self.forward(claim_emb, avail)
            return logits.argmax(dim=-1)

# Entropy regularisation helper
def srp_loss(logits, targets, beta=0.05):
    ce   = F.cross_entropy(logits, targets)
    probs = F.softmax(logits, dim=-1)
    ent  = -(probs * probs.log().clamp(min=-1e9)).sum(dim=-1).mean()
    return ce - beta * ent  # maximise entropy → diverse routing

3,457 params each · 5 networks · Paper §4.4
# ── Stage 3a: Sensor-Specific Verdict Networks ────────────────────────────
# vⱼ = Vᵢ(xᵢ,ⱼ)  where i = π̂(cⱼ)
# One dedicated 2-layer MLP per sensor — trained ONLY on claims
# where that sensor is authoritative. This is the key design choice:
# non-authoritative sensors produce verdicts near 0.5 (uncertain),
# degrading gracefully under routing errors.

class VerdictNetwork(nn.Module):
    """
    Binary hallucination classifier for a single sensor modality.
    Input : sensor feature vector (shape: SENSOR_FEAT_DIM)
    Output: scalar ∈ [0,1]  (1 = consistent, 0 = hallucinated)
    """

    def __init__(self, feat_dim: int = 32, hidden: int = 32):
        super().__init__()
        self.net = nn.Sequential(
            nn.Linear(feat_dim, hidden),
            nn.GELU(),
            nn.Linear(hidden, 1),
            nn.Sigmoid(),
        )

    def forward(self, sensor_feat):
        return self.net(sensor_feat).squeeze(-1)


# Signal-attenuation model (paper §4.4)
# Non-authoritative features are attenuated toward 0.5,
# ensuring routing errors produce uncertain (not confident wrong) verdicts.
def get_sensor_features(scene, sensor_idx, claim_idx, snr_matrix):
    base_feat = scene["sensor_data"][sensor_idx][claim_idx]
    snr       = snr_matrix[claim_idx][sensor_idx]
    # Attenuate non-authoritative features toward decision boundary
    attenuation = 1.0 - (1.0 / (1.0 + snr))
    return base_feat * attenuation + (1 - attenuation) * 0.5


# Build all 5 verdict networks (one per sensor)
verdict_nets = [VerdictNetwork(feat_dim=32) for _ in range(5)]
# Total verdict params: 5 × 3,457 = 17,285
contrastive_reweighting.py
λ=2.5 default · Paper §4.5, Eq. 9
# ── Stage 3b: Contrastive Re-weighting ───────────────────────────────────
# Ψ(y, X_S) = (1/n) Σⱼ (2·vⱼ − 1) · 𝟙[cⱼ ∈ E(y)]
# p_SB(y | x_rgb, X_S) ∝ p_M(y | x_rgb) · exp(λ · Ψ)
#
# Bernoulli suppression at claim level:
# P(suppress cⱼ) = σ(λ · (0.5 − vⱼ))
# → high vⱼ (sensor-consistent) → low suppression prob
# → low vⱼ (hallucinated)       → high suppression prob

import torch, numpy as np

class ContrastiveReweighter:
    def __init__(self, lam: float = 2.5):
        self.lam = lam

    def suppress(self, verdicts, hallucinated_gt, rng):
        """
        verdicts       : (n_claims,) float tensor in [0,1]
        hallucinated_gt: (n_claims,) bool  — ground truth labels
        rng            : numpy RandomState for reproducibility
        Returns corrected hallucination labels after suppression.
        """
        corrected = hallucinated_gt.copy()
        for j, v in enumerate(verdicts):
            # Suppression probability (logistic)
            p_sup = 1.0 / (1.0 + np.exp(-self.lam * (0.5 - v)))
            if hallucinated_gt[j]:
                # Suppress hallucinated claim with prob p_sup
                if rng.rand() < p_sup:
                    corrected[j] = False
        return corrected

    def scene_consistency_score(self, verdicts):
        """Ψ(y, X_S) — Eq. 8 in the paper."""
        if len(verdicts) == 0:
            return 0.0
        return np.mean([2 * v - 1 for v in verdicts])

# λ sensitivity: lower λ → conservative, higher λ → aggressive suppression
# Paper reports: CHAIR_S 43.7 at λ=0.5 → 18.1 at λ=5.0 (monotone ↓)

# ── Joint Training: SRP + Verdict Networks ────────────────────────────────
# All 5 verdict nets + SRP trained jointly with Adam
# Cosine annealing LR schedule over 80 epochs

optimizer = torch.optim.Adam(
    list(srp.parameters()) +
    [p for vn in verdict_nets for p in vn.parameters()],
    lr=3e-3, weight_decay=1e-4
)
scheduler = torch.optim.lr_scheduler.CosineAnnealingLR(
    optimizer, T_max=80, eta_min=1.5e-4
)

for epoch in range(80):
    srp.train()
    for vn in verdict_nets: vn.train()
    epoch_loss = 0.0

    for batch in train_loader:
        claim_emb, avail, route_tgt = (
            batch["claim_emb"], batch["avail"], batch["route_target"]
        )
        # SRP forward + loss
        srp_logits = srp(claim_emb, avail)
        loss_srp   = srp_loss(srp_logits, route_tgt, beta=0.05)

        # Verdict network losses (sensor-specific)
        loss_verd = 0.0
        for si in range(N_SENSORS):
            mask = route_tgt == si
            if mask.any():
                feats   = batch[f"sensor_feat_{si}"][mask]
                targets = batch["verdict_target"][mask].float()
                preds   = verdict_nets[si](feats)
                loss_verd += F.binary_cross_entropy(preds, targets)

        loss = loss_srp + loss_verd
        optimizer.zero_grad()
        loss.backward()
        optimizer.step()
        epoch_loss += loss.item()

    scheduler.step()

# ── Evaluation Metrics (Paper §6) ─────────────────────────────────────────

def compute_chair_s(corrected_labels_list):
    """CHAIR_S: % of scenes with ≥1 remaining hallucinated claim (↓)."""
    hall = sum(any(labels) for labels in corrected_labels_list if labels)
    return 100.0 * hall / max(len(corrected_labels_list), 1)

def compute_chair_i(corrected_labels_list):
    """CHAIR_I: % of individual claims still hallucinated (↓)."""
    flat = [l for labels in corrected_labels_list for l in labels]
    return 100.0 * sum(flat) / max(len(flat), 1)

def compute_sh_f1(corrected_list, gt_list):
    """SH-F1: harmonic mean of suppression precision and recall (↑)."""
    tp = fp = fn = 0
    for corr, gt in zip(corrected_list, gt_list):
        for c, g in zip(corr, gt):
            suppressed = g and not c
            if g and suppressed:  tp += 1
            elif not g and suppressed: fp += 1
            elif g and not suppressed: fn += 1
    prec = tp / max(tp + fp, 1)
    rec  = tp / max(tp + fn, 1)
    return 100.0 * 2 * prec * rec / max(prec + rec, 1e-9)
